# ETH/USDT 趋势波动率交易策略

## 策略概述

这是一个基于趋势跟踪和波动率过滤的ETH/USDT期货交易策略。该策略结合了以下核心元素：

- **趋势判断**：使用EMA交叉确认市场趋势方向
- **波动率过滤**：基于ATR指标识别市场波动状况，避免在极端波动时入场
- **风险管理**：动态仓位管理和严格的止损策略
- **交易统计**：完整的交易记录和绩效分析

## 安装依赖

```bash
pip install requests pandas numpy
```

## 使用方法

### 1. 配置API密钥

在使用前，您需要配置Binance API密钥。有两种方式：

#### 方法一：环境变量（推荐）

在Windows命令提示符中设置环境变量：

```cmd
setx BINANCE_API_KEY "您的API密钥"
setx BINANCE_SECRET_KEY "您的密钥"
```

#### 方法二：直接修改代码

在`trendVolatilityStrategy.py`文件中，找到以下行并替换为您的密钥：

```python
self.api_key = os.environ.get('BINANCE_API_KEY', 'your_api_key_here')
self.secret_key = os.environ.get('BINANCE_SECRET_KEY', 'your_secret_key_here')
```

### 2. 策略参数设置

您可以在初始化策略时调整以下关键参数：

| 参数 | 说明 | 默认值 | 推荐范围 |
|------|------|--------|----------|
| symbol | 交易对 | 'ETHUSDT' | 保持默认 |
| timeframe | K线周期 | '1h' | '15m', '1h', '4h' |
| ema_short | 短期EMA周期 | 12 | 8-21 |
| ema_long | 长期EMA周期 | 26 | 21-55 |
| atr_period | ATR计算周期 | 14 | 14-21 |
| risk_per_trade | 单笔交易风险比例 | 0.02 | 0.01-0.05 |
| stop_loss_pct | 固定止损比例 | 0.05 | 0.03-0.10 |

### 3. 运行策略

```bash
python trendVolatilityStrategy.py
```

## 策略核心逻辑

### 入场条件

1. **多头入场**：
   - 短期EMA上穿长期EMA（金叉）
   - 当前ATR值在正常范围内（不是高波动状态）
   - 当前无持仓

2. **空头入场**：
   - 短期EMA下穿长期EMA（死叉）
   - 当前ATR值在正常范围内（不是高波动状态）
   - 当前无持仓

### 出场条件

1. **止盈**：价格达到预设止盈目标（基于ATR计算）
2. **止损**：价格触及止损位（固定百分比或ATR倍数）
3. **趋势反转**：EMA交叉方向改变

### 仓位管理

仓位大小基于以下因素动态计算：
- 账户余额
- 风险百分比设置
- 当前ATR值（波动率）
- 单笔交易最大资金比例限制（20%）

## 风险提示

1. **API安全**：确保API密钥仅具有交易权限，不要授予提现权限
2. **资金管理**：建议先用小资金测试，熟悉策略表现
3. **市场风险**：加密货币市场波动剧烈，策略可能在极端市场条件下表现不佳
4. **技术风险**：确保网络连接稳定，避免在策略运行时中断
5. **参数调整**：根据市场状况定期评估和调整策略参数

## 交易记录

策略会自动记录交易历史到JSON文件，格式为：`trade_history_{symbol}_{date}.json`

## 优化建议

1. **参数优化**：可以通过`optimize_parameters`方法测试不同参数组合
2. **多时间框架分析**：结合多个时间周期的信号提高准确性
3. **滑点和手续费**：在回测中考虑真实交易成本
4. **监控系统**：添加异常监控和通知机制

## 注意事项

- 该策略目前仅支持ETH/USDT期货交易对
- 运行策略需要稳定的网络连接和足够的账户余额
- 定期备份交易记录数据
- 实盘交易前务必先进行充分的回测验证
